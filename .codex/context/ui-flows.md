# UI Flows

## Homepage Location Flow

Homepage lets the user:

- search for a location
- choose from Google Maps / places
- use current location
- persist the location to backend only when logged in and phone exists

If not logged in, the chosen location is only local browser state.

## Customer Mode

Main customer surfaces:

- homepage
- restaurants
- categories
- menu items
- cart
- wishlist
- profile
- reservations

## Merchant Mode

Merchant portal is inside this same repo.

Current merchant surface includes:

- merchant landing/dashboard
- restaurant context switching
- restaurant presence / catalog / reservation / promo entry points

Long term idea:

- same account can operate in customer mode and merchant mode
- workspace choice should come from backend-controlled capabilities
